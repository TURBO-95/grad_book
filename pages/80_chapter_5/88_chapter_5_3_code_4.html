4. Asynchronous Agent Streaming Loop

Extracted from ai_engine/agent_runner.py, this generator dictates the LLM reasoning cycle. It demonstrates how exceptions are utilized for flow-control, catching `ActionRequiresConfirmationError` to suspend the LLM and serialize the `_resume_state`.

async def stream_chat(self, messages: List[BaseMessage]) -> AsyncGenerator[Dict, None]: try: # Stream raw generation tokens to client async for chunk in self.llm_with_tools.astream(messages): yield { "type": "token", "content": chunk.content } # Process Tool/Action invocations for tool_call in ai_message_chunk.tool_calls: try: # Execution raises exception if gated by config result = await tool.ainvoke(tool_call["args"]) except ActionRequiresConfirmationError as e: # Halt execution and serialize context for DB storage yield { "type": "pause", "reason": "confirmation_required", "action_name": e.action_name, "params": e.params, "_resume_state": { "assistant_message": ai_message_chunk, "completed_tool_results": tool_results } } return # Exit generator completely until human approval finally: set_correlation_id(None)
