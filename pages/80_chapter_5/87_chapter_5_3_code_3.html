3. Non-Blocking Async WebSocket Chat Loop

Extracted from apis/v1/chat.py, this snippet orchestrates the FastAPI persistent connection. Crucially, it highlights how context-editing triggers a cascade deletion of database messages prior to re-launching the asynchronous generation task.

@router.websocket("/ws") async def chat_websocket(websocket: WebSocket): await websocket.accept() gateway = AIGatewayService() active_generation_task = None while True: msg = await websocket.receive_json() msg_type = msg.get("type") if msg_type == "edit": # Safely cancel the active thread if generating if active_generation_task and not active_generation_task.done(): cancel_token.set() # Cascade deletion logic to reset conversational context async with AsyncSessionLocal() as db: await gateway.edit_message_cascade( db, conversation_id, UUID(msg.get("message_id")), msg.get("content") ) messages = await gateway.load_message_history(db, conversation_id) await websocket.send_json({"type": "edit_successful"}) # Re-trigger LLM evaluation with sanitized historical context active_generation_task = asyncio.create_task(run_ai_stream(messages))
