Function Calling and Tool Use via JSON Schemas

While ReAct provides the cognitive loop, the mechanical bridge between a probabilistic language model and a deterministic software API requires a rigid data structure. Enterprise APIs (REST, gRPC, etc.) do not accept natural language; they require strictly typed, meticulously formatted payloads (most commonly JSON). To facilitate this, foundational model developers explicitly fine-tune LLMs for Function Calling (or Tool Use).

The orchestrating application provides the LLM with a system prompt containing an array of available tools, defined using standard JSON Schema syntax. The LLM is trained to parse this schema and, instead of generating text, output a perfectly structured JSON string containing the exact arguments required by the target API.

By treating the LLM as a highly advanced text-to-JSON compiler, developers can safely map probabilistic human intent into executable code. If a required parameter is missing, the LLM recognizes the omission in the schema and naturally asks the user for clarification before formulating the final API call.
