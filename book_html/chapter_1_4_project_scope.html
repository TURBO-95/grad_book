1.4 Project Scope

The scope of the IntgraServeAI project defines the explicit boundaries of the deliverables, outlining the specific features and functionalities that will be developed, as well as those deliberately excluded to maintain project focus.

In-Scope: The project encompasses the design, development, and testing of the core IntgraServeAI backend service, primarily utilizing Python. It includes the realization of the RAG pipeline for document ingestion and semantic search, alongside the dynamic Action Engine capable of translating abstract LLM intents into executable REST, gRPC, or GraphQL requests. The scope also covers the development of a functional User Interface (UI) dashboard to demonstrate the agent's capabilities in real-time. Finally, the project includes the delivery of comprehensive system architecture documentation, API reference guides, and deployment blueprints utilizing containerization (Docker).

Out-of-Scope: The project will not deliver a commercially hosted, multi-tenant SaaS application. Advanced enterprise features such as global compliance certifications (e.g., SOC2, GDPR compliance auditing), high-availability production load balancing, and integrated billing modules fall outside the academic scope of this graduation project. Additionally, while the system automates tasks, it is not designed to permanently replace human support agents; rather, it acts as an intelligent intermediary, though human-in-the-loop escalation UI falls outside the primary prototype scope.
